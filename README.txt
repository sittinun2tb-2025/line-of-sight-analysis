# prepare_utm_fixed_geojson.py
1) โหลดข้อมูล bkk_footprints.geojson มาเก็บไว้ก่อน
2) เคลียร์ข้อมูลใหม่ได้ bkk_footprints_utm_fixed.geojson โดยรัน py

#
1. Observer ยืนอยู่ตรงนั้นได้จริงหรือไม่ — 44.7% ของ cell อยู่ในตัวตึก
จากการตรวจ: 876 จาก 1,961 cells มีจุด center อยู่ข้างใน footprint อาคาร — โมเดลปัจจุบัน
นับ cell พวกนี้เป็น "จุดยืนมอง" ทั้งที่คนเดินถนนยืนตรงนั้นไม่ได้ (มันคือกลางตึก) สถิติ "19% มองเห็น" 
จึงเจือจางด้วยจุดที่ไม่มีวันมีคนยืนจริง ถ้ากรองเหลือเฉพาะพื้นที่สาธารณะ (ถนน ทางเท้า ลานโล่ง) 
สัดส่วนและแผนที่จะเปลี่ยนความหมายไปมาก — นี่คือประเด็นที่ควรแก้ก่อนนำตัวเลขไปอ้างอิง 
แก้ได้ 2 ระดับ: ง่ายสุดคือ mask cell ที่อยู่ใน footprint ออก / ดีสุดคือใช้ชั้นข้อมูลถนน-พื้นที่สาธารณะ
เป็นตัวกำหนดจุด observer แทน grid ล้วน ๆ

2. การมองเห็น "ขากลับ" และบริบทของแหล่ง (setting)
งานอนุรักษ์ตามแนว ICOMOS/HIA ไม่ได้สนใจแค่ "คนเห็นโบราณสถานไหม" แต่รวมถึง 
วิวจากตัวโบราณสถานออกไป (เช่น มองจากยอดเจดีย์แล้วเห็นตึกสูงโผล่หลังแนวหลังคาเดิมไหม) 
และ intervisibility ระหว่างโบราณสถานหลายแห่ง (แนวมองระหว่างวัด-วัง-แม่น้ำ ที่เป็น
องค์ประกอบเชิงผังของเมืองเก่า) — โมเดลของคุณคำนวณกลับทิศได้อยู่แล้วแค่สลับบทบาทจุด

3. ความไม่แน่นอนของผล (uncertainty) ควรถูกรายงานคู่กับแผนที่
เราเห็นแล้วว่า cell จำนวนมากแพ้-ชนะกันหลัก เศษเมตร (grid 412 ต่างกัน 26 ซม.!) — 
ถ้าความสูงตึก (AGL) คลาดเคลื่อน ±1-2 ม. ซึ่งเป็นเรื่องปกติของข้อมูลแบบนี้ ผลจะพลิกเป็นร้อย cell 
แนวทางที่ดีคือทำ sensitivity analysis (เช่น รันซ้ำโดยสุ่ม AGL ±2 ม. หลาย ๆ รอบ แล้วรายงาน 
"ความน่าจะเป็นที่มองเห็น" ต่อ cell แทน 0/1 แข็ง ๆ) — แผนที่แบบ probabilistic ปกป้องคุณจากการ
ถูกโต้แย้งได้ดีกว่าแผนที่ binary มาก

4. มุมมองแบบเคลื่อนที่ (serial vision)
คนไม่ได้ยืนดูจากจุดเดียว — ประสบการณ์จริงคือการเห็นตามลำดับระหว่างเดินเข้าหา (เดินจากท่าเรือ/ป้ายรถเมล์เข้าสู่แหล่ง) 
การวิเคราะห์ visibility ตามแนวเส้นทางเข้าถึงหลัก (ทุก ๆ 10 ม. บนถนน) ให้คำตอบเชิงอนุรักษ์คนละแบบกับ grid รอบทิศ 
และมักตรงกับที่คณะกรรมการอนุรักษ์ใช้ตัดสินจริง

5. "เห็น" ≠ "เห็นอย่างมีคุณค่า"
ระยะ 250 ม. เห็นยอดเจดีย์โผล่ 1 องศาเหนือหลังคาตึกก็นับ visible=1 เท่ากับยืนหน้าแหล่งเต็มตา — 
มิติที่ยังไม่ได้วัด: มุมมองเชิงทัศน์ (visual angle ที่โบราณสถานกินในสายตา), ฉากหลัง (เห็นตัดกับฟ้า vs จมในตึกกระจก), 
เวลากลางคืน/แสงส่อง — weight จากระยะทางเป็น proxy เริ่มต้นที่ดี แต่ถ้ารายงานจะใช้คำว่า "คุณภาพการมองเห็น" 
ควรอัปเกรดเป็น visual angle จริงซึ่งคำนวณได้จากข้อมูลที่มีอยู่แล้ว

6. มิติเวลา
ข้อมูลตึกคือ snapshot ณ วันเก็บข้อมูล — มีตึกใหม่กำลังสร้าง/เครนชั่วคราวไหม, ต้นไม้ผลัดใบตามฤดูเปลี่ยนการบดบัง, 
และสำคัญสุดเชิงนโยบาย: ผลกระทบสะสม (cumulative) — โครงการเดี่ยว ๆ อาจไม่บัง แต่หลายโครงการรวมกันปิด corridor ได้ 
ควรมี scenario "อนาคตตามสิทธิ์การพัฒนาเต็มเพดานผังเมือง" ไม่ใช่แค่สภาพปัจจุบัน

7. กรอบกฎหมายที่ผลวิเคราะห์ต้องไปเทียบ
ถ้าพื้นที่ศึกษาอยู่เขตรัตนโกสินทร์/ใกล้โบราณสถานขึ้นทะเบียน มีข้อบัญญัติควบคุมความสูงอาคารเป็นชั้น ๆ อยู่แล้ว — 
การวางผล LOS ซ้อนกับแนวเขตควบคุมตามกฎหมายปัจจุบันจะชี้ได้ทันทีว่า จุดไหนที่กฎหมายปัจจุบัน "คุ้มครองไม่พอ" 
กับ corridor ที่พบ ซึ่งนั่นคือ output ที่มีพลังเชิงนโยบายที่สุดของงานนี้
